# PCBuilder

PCBuilder is a Unity-based interactive training prototype for PC assembly and maintenance. The current module teaches, practices, and assesses a dual-channel RAM installation procedure on a simplified motherboard workbench.

The project focuses on hands-on procedural training rather than a static tutorial. Learners open RAM slot retaining clips, pick up RAM modules, rotate them into the correct orientation, install them into the correct dual-channel slots, and verify the completed procedure. The simulation records errors, hints, timing, score, and a chronological action log, then exports session reports.

## Features

- Three training modes: Teach, Practice, and Assess.
- Guided five-step dual-channel RAM installation workflow.
- Mouse-driven part selection, drag-and-drop installation, and camera orbit controls.
- Slot compatibility checks for correct module, correct slot, retaining clip state, proximity, and orientation.
- Visual highlights for valid, invalid, warning, hover, and selected states.
- Score tracking with penalties for wrong slots, orientation mistakes, out-of-order actions, and hint usage.
- Automatic JSON and HTML report generation at the end of a completed training run.
- Programmatic TextMesh Pro UI, so the main interface is generated by code at runtime.
- Editor scene validation utility for checking required objects, components, IDs, colliders, and snap point setup.

## Project Status

## Getting Started

1. Install Unity Hub and Unity Editor `6000.5.0f1`.
2. Install Git LFS if it is not already installed:

   ```bash
   git lfs install
   ```

3. Clone your fork:

   ```bash
   git clone <your-fork-url>
   cd PCBuilder
   git lfs pull
   ```

4. Open the project from Unity Hub:

   - Select **Add project from disk**.
   - Choose the cloned `PCBuilder` folder.
   - Open it with Unity `6000.5.0f1`.

5. Open the prototype scene:

   ```text
   Assets/Scenes/PCBuilderPrototype.unity
   ```

6. Press **Play** in the Unity Editor.

7. Choose a mode from the start menu and click **Start Procedure**.

## How to Play

The RAM installation procedure is:

1. Open the retaining clip for `RamSlot_A2`.
2. Open the retaining clip for `RamSlot_B2`.
3. Install `RamModule_1` into `RamSlot_A2`.
4. Install `RamModule_2` into `RamSlot_B2`.
5. Verify that both RAM modules are installed and locked.

### Training Modes

| Mode | Behavior |
| --- | --- |
| Teach | Shows full instructions and highlights the current object and target slot. |
| Practice | Shows current instructions and allows hints, but reduces active guidance. |
| Assess | Hides hints and expects the learner to complete the procedure with minimal support. |

### Controls

| Control | Action |
| --- | --- |
| Left mouse button | Select, hold, drag, and release RAM modules. |
| Click retaining clip | Open the current required slot clip. |
| Q / E while dragging | Rotate the held RAM module around its local Y axis. |
| Escape while dragging | Cancel the active drag and return the part to its start position. |
| Right mouse button + drag | Orbit the camera around the motherboard workspace. |
| Mouse wheel | Zoom the camera in or out. |
| F | Refocus the camera on the workbench. |

## Scoring and Reports

Sessions start at `100` points. The scoring service applies penalties as follows:

| Event | Penalty |
| --- | ---: |
| Wrong slot | -8 |
| Out-of-order action | -5 |
| Orientation error | -4 |
| Hint used | -3 |

A score of `70` or higher is considered passing.

When the procedure is completed, PCBuilder writes both a JSON report and an HTML report to Unity's persistent data directory:

```text
<Application.persistentDataPath>/Reports/
```

On Windows with the current project settings, this is typically:

```text
%USERPROFILE%\AppData\LocalLow\DefaultCompany\PCBuilder\Reports\
```

Reports include procedure name, mode, date/time, score, pass/fail status, duration, mistake count, hint count, completed steps, and the chronological action log.

## Architecture Overview

PCBuilder is organized around small components with clear responsibilities:

- `ProcedureRunner` owns the active training mode, step list, state progression, validation, feedback events, reset flow, and report trigger.
- `ProcedureData` defines reusable procedure and step data structures.
- `ScoringService` tracks elapsed time, correct actions, mistakes, hints, score, pass/fail status, and action logs.
- `InteractionController` performs mouse raycasts, manages held objects, validates releases against slots, and routes clip clicks.
- `DraggablePart` handles pickup, drag movement, Q/E rotation, cancellation, and return-to-start behavior.
- `InstallablePart` stores part identity, snapping behavior, locking, and reset behavior.
- `InstallationSlot` validates accepted component ID, clip state, distance tolerance, and orientation tolerance.
- `RamSlotClip` represents interactive retaining clips for RAM slots.
- `HighlightController` manages hover, selected, valid, invalid, and warning material states.
- `OrbitCamera` provides workbench-focused orbit, zoom, and focus controls.
- `PCBuilderUI` creates the runtime Canvas, mode menu, procedure HUD, controls, progress bar, and completion panel.
- `ReportService` exports machine-readable JSON and human-readable HTML session reports.
- `SceneValidationUtility` adds **Tools > PCBuilder > Validate Scene** in the Unity Editor.

## Scene Validation

The editor validation tool helps confirm the prototype scene is configured correctly.

To run it:

1. Open `Assets/Scenes/PCBuilderPrototype.unity`.
2. In the Unity menu bar, choose **Tools > PCBuilder > Validate Scene**.
3. Check the Unity Console for any reported errors or warnings.

The validator checks for required scene objects, required runtime components, unique IDs, kinematic Rigidbody settings, trigger colliders, snap point references, Canvas presence, and EventSystem presence.

## Extending the Simulator

### Add a New Installable Part

1. Add a mesh or primitive to the scene.
2. Attach:
   - `SelectableObject`
   - `DraggablePart`
   - `HighlightController`
   - `InstallablePart`
3. Configure a unique component ID on `InstallablePart`.
4. Add a `Rigidbody` with `Is Kinematic` enabled and `Use Gravity` disabled.
5. Add a collider suitable for raycasting and overlap checks.

### Add a New Slot

1. Create a slot GameObject.
2. Attach:
   - `InstallationSlot`
   - `HighlightController`
   - `BoxCollider`
3. Mark the `BoxCollider` as `Is Trigger`.
4. Create and assign a child `SnapPoint` transform.
5. Set the slot ID and accepted component ID.
6. Assign any related retaining clip if the part requires one.

### Add or Change Procedure Steps

Procedure steps can be supplied through a `ProcedureDefinition` ScriptableObject or through the fallback step list in `ProcedureRunner.InitializeProcedure()`.

Each step defines:

- `stepId`
- `instruction`
- `hint`
- `actionType`
- `objectId`
- `targetId`
- prerequisite step IDs
- optional error feedback

The procedure runner validates every interaction against the current step before allowing progress.

## Troubleshooting

### The scene opens but nothing interactive appears

Confirm that `Assets/Scenes/PCBuilderPrototype.unity` is open. `SampleScene.unity` is a basic Unity sample scene.

### UI clicks or mouse input do not work

Confirm the project is using the Input System package and that Unity has restored packages from `Packages/manifest.json`. The UI creates an EventSystem automatically if one is missing.

### Reports are not visible in the project folder

Reports are written to `Application.persistentDataPath`, not to `Assets/`. Check the `Reports` folder under the persistent data path shown in the completion screen or Unity Console.

### Validation reports missing scene objects

Open `PCBuilderPrototype.unity` before running **Tools > PCBuilder > Validate Scene**. The validator checks the active scene hierarchy.
